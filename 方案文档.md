# 个人网站建设方案

> 目标：在 GitHub Pages 上搭建个人网站，包含个人介绍、博客、随笔、照片墙、基金展示，后续可迁移至自有服务器。

---

## 一、技术选型

| 类别 | 技术 | 理由 |
|------|------|------|
| 框架 | Next.js 14（App Router）+ TypeScript | 生态成熟，静态导出兼容 GitHub Pages，后续迁移服务器可直接开启 SSR |
| 样式 | Tailwind CSS | 原子类开发效率高，易于维护主题 |
| UI 风格 | 参考 uiverse.io 卡片组件 | 平和好看，hover 动效丰富 |
| 内容管理 | Markdown 文件 + gray-matter + remark | 直接写 `.md` 提交 Git，无需后台系统 |
| 图片 | `next/image` + `/public` 目录 | 自动优化，静态导出时设置 `unoptimized: true` |
| 部署 | GitHub Actions → GitHub Pages | 推送代码自动构建发布，后续可配置自定义域名 |

## 三、页面规划

### 1. 首页 `/`
- **Hero 区**：头像、姓名、一句话介绍、社交链接（GitHub / 邮件）
- **最近博客**：最新 3 篇博客卡片，带 hover 动效
- **最近随笔**：最新 2 篇随笔预览
- **快速入口**：导航至照片墙、基金页

### 2. 博客 `/blog`
- 卡片列表，展示标题、日期、标签、摘要
- 支持按标签筛选（可选，二期实现）
- 详情页：Markdown 渲染，代码高亮，估读时间

### 3. 随笔 `/essays`
- 比博客更轻松的排版风格
- 卡片列表 → 详情页
- 可附带照片

### 4. 照片墙 `/photos`
- 网格/瀑布流布局
- 悬停显示照片描述
- 点击放大预览（lightbox 效果）

### 5. 基金 `/fund`
- 当前：占位卡片，显示"功能建设中"
- 后续扩展方向：
  - 手动维护关注基金列表（静态数据）
  - 接入基金净值 API 展示实时数据
  - 发布基金分析文章

### 6. 导航栏
- 固定顶部，链接：首页 / 博客 / 随笔 / 照片 / 基金
- 移动端汉堡菜单
- 可选深色模式切换按钮

## 四、UI 设计规范

### 配色方案（平和系）

| 角色 | 色值 | 用途 |
|------|------|------|
| 背景 | `#FAFAF9`（暖白） | 页面背景 |
| 表面 | `#FFFFFF` | 卡片背景 |
| 边框 | `#E5E7EB` | 卡片边框、分割线 |
| 主文字 | `#1C1917` | 标题、正文 |
| 次文字 | `#78716C` | 日期、标签、描述 |
| 强调色 | `#6366F1`（柔和靛蓝） | 链接、按钮、hover 状态 |

### 卡片风格（参考 uiverse.io）
- 圆角：`rounded-2xl`
- 阴影：默认 `shadow-sm`，hover 时 `shadow-md` + 轻微上移（`-translate-y-1`）
- 过渡：`transition-all duration-300`
- 边框：`border border-stone-200`，hover 时 `border-indigo-200`

### 字体
- 中文：思源宋体 / Noto Serif SC（正文优雅）
- 代码：JetBrains Mono

## 六、部署流程

### GitHub Pages 部署

1. 创建 GitHub 仓库（推荐命名 `username.github.io` 或 `my-site`）
2. 配置 `next.config.js`：
   ```js
   // 若仓库名不是 username.github.io，需要设置 basePath
   basePath: '/仓库名',
   output: 'export',
   images: { unoptimized: true }
   ```
3. 添加 `.github/workflows/deploy.yml`，push main 分支时自动构建并发布到 `gh-pages` 分支
4. GitHub 仓库设置 → Pages → Source 选择 `gh-pages` 分支

### 迁移到自有服务器（未来）
- 去掉 `output: 'export'`，恢复完整 SSR 能力
- 用 Nginx 反向代理 `next start`，或用 Docker 部署
- 可选配置自定义域名 + HTTPS（Certbot）
